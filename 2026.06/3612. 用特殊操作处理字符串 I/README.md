# 3612. 用特殊操作处理字符串 I

题目链接：[LeetCode 3612. 用特殊操作处理字符串 I](https://leetcode.cn/problems/process-string-with-special-operations-i/)

## 题目描述

给你一个字符串 `s`。字符串中包含小写英文字母，以及三个特殊字符：

```text
*
#
%
```

从左到右处理 `s`，用一个结果字符串 `ans` 保存当前处理出来的内容。

处理规则如下：

| 当前字符 `ch` | 操作 |
|---|---|
| 小写字母 | 把 `ch` 追加到 `ans` 的末尾 |
| `*` | 删除 `ans` 的最后一个字符；如果 `ans` 为空，则什么也不做 |
| `#` | 把当前 `ans` 复制一份，并追加到 `ans` 后面 |
| `%` | 反转当前 `ans` |

处理完整个字符串后，返回最终的 `ans`。

## 核心思路

这道题最直接的做法就是模拟。

我们定义一个变量：

```cpp
string ans;
```

它表示“当前已经处理出来的字符串”。

然后从左到右遍历原字符串 `s`：

```cpp
for (char ch : s)
```

每次取出一个字符 `ch`，根据 `ch` 的类型修改 `ans`。

这种题的重点不是复杂算法，而是要把每个特殊字符的含义写准确，尤其是：

1. `*` 删除最后一个字符之前，要判断 `ans` 是否为空。
2. `#` 是复制“当前完整的 `ans`”，不是复制最后一个字符。
3. `%` 是反转“当前完整的 `ans`”，不是反转原字符串 `s`。

## 代码实现

核心代码见 [solution.cpp](D:\leetcode\2026.6\3612. 用特殊操作处理字符串 I\solution.cpp)。

LeetCode 提交时只需要提交下面这个 `Solution` 类：

```cpp
class Solution {
public:
    string processStr(string s) {
        string ans;

        for (char ch : s) {
            if (ch >= 'a' && ch <= 'z') {
                ans.push_back(ch);
            } else if (ch == '*') {
                if (!ans.empty()) {
                    ans.pop_back();
                }
            } else if (ch == '#') {
                ans += ans;
            } else if (ch == '%') {
                reverse(ans.begin(), ans.end());
            }
        }

        return ans;
    }
};
```

注意：需要包含头文件：

```cpp
#include <algorithm>
#include <string>
```

其中：

- `string` 用于保存 `ans`
- `reverse` 来自 `<algorithm>`

## 变量含义

代码中主要有两个变量：

```cpp
string ans;
char ch;
```

它们分别表示：

| 变量 | 含义 |
|---|---|
| `ans` | 当前已经处理出来的结果字符串 |
| `ch` | 当前正在处理的字符 |

在每一轮循环中，代码会根据 `ch` 修改 `ans`。

比如：

```cpp
if (ch >= 'a' && ch <= 'z') {
    ans.push_back(ch);
}
```

如果 `ch` 是小写字母，就把它追加到 `ans` 的末尾。

```cpp
else if (ch == '*') {
    if (!ans.empty()) {
        ans.pop_back();
    }
}
```

如果 `ch` 是 `*`，就删除 `ans` 的最后一个字符。

这里必须写：

```cpp
if (!ans.empty())
```

因为如果 `ans` 已经是空字符串，再执行 `pop_back()` 会出错。

```cpp
else if (ch == '#') {
    ans += ans;
}
```

如果 `ch` 是 `#`，就把当前 `ans` 复制一遍。

例如当前：

```text
ans = "ab"
```

执行后：

```text
ans = "abab"
```

```cpp
else if (ch == '%') {
    reverse(ans.begin(), ans.end());
}
```

如果 `ch` 是 `%`，就把当前 `ans` 反转。

例如当前：

```text
ans = "abc"
```

执行后：

```text
ans = "cba"
```

## 正确性说明

我们证明代码最终返回的 `ans` 就是题目要求的处理结果。

处理字符串时，代码从左到右遍历：

```cpp
for (char ch : s)
```

设已经处理完前 `i` 个字符后，变量 `ans` 等于题目规则处理前 `i` 个字符后得到的结果。

现在看第 `i + 1` 个字符，也就是代码中的 `ch`。

如果 `ch` 是小写字母，题目要求把它追加到结果末尾，代码执行：

```cpp
ans.push_back(ch);
```

两者一致。

如果 `ch` 是 `*`，题目要求删除结果末尾字符；如果结果为空则不操作。代码执行：

```cpp
if (!ans.empty()) {
    ans.pop_back();
}
```

两者一致。

如果 `ch` 是 `#`，题目要求复制当前结果并追加。代码执行：

```cpp
ans += ans;
```

两者一致。

如果 `ch` 是 `%`，题目要求反转当前结果。代码执行：

```cpp
reverse(ans.begin(), ans.end());
```

两者一致。

因此每处理一个字符后，代码中的 `ans` 都和题目规则得到的结果一致。

当所有字符都处理完后，`ans` 就是最终答案，所以返回 `ans` 是正确的。

## 复杂度分析

设 `n` 是输入字符串 `s` 的长度，`L` 是处理过程中 `ans` 的最大长度。

时间复杂度：

```text
O(n * L)
```

原因：

- 遍历 `s` 需要 `O(n)`
- 小写字母追加通常是 `O(1)`
- `*` 删除末尾是 `O(1)`
- `#` 复制当前 `ans`，可能需要 `O(L)`
- `%` 反转当前 `ans`，可能需要 `O(L)`

所以整体可以写成 `O(n * L)`。在题目 I 的规模下，直接模拟足够通过。

空间复杂度：

```text
O(L)
```

原因是我们需要用 `ans` 保存当前结果字符串。

## 常见错误

### 错误 1：`*` 时没有判断空字符串

错误写法：

```cpp
ans.pop_back();
```

如果当前 `ans` 是空字符串，这样会出问题。

正确写法：

```cpp
if (!ans.empty()) {
    ans.pop_back();
}
```

### 错误 2：把 `#` 理解成复制最后一个字符

`#` 复制的是整个当前 `ans`。

如果：

```text
ans = "abc"
```

遇到 `#` 后：

```text
ans = "abcabc"
```

不是：

```text
ans = "abcc"
```

### 错误 3：把 `%` 理解成反转剩余字符串

`%` 反转的是当前已经处理出来的 `ans`。

它不会影响还没有遍历到的字符。

例如：

```text
s = "ab%c"
```

处理到 `%` 时：

```text
ans = "ab"
```

反转后：

```text
ans = "ba"
```

然后继续处理后面的 `'c'`：

```text
ans = "bac"
```

### 错误 4：手算时忘记操作顺序

所有字符必须从左到右依次处理。

比如：

```text
s = "abc%#*"
```

必须先把 `"abc"` 反转成 `"cba"`，再复制成 `"cbacba"`，最后删除末尾得到 `"cbacb"`。

不能先复制 `"abc"`，再反转。

## 总结

这道题的关键是维护一个当前结果 `ans`，并按照字符 `ch` 的类型进行模拟：

1. 小写字母：`ans.push_back(ch)`
2. `*`：如果 `ans` 非空，则 `ans.pop_back()`
3. `#`：`ans += ans`
4. `%`：`reverse(ans.begin(), ans.end())`

题目 I 的数据规模适合直接模拟，代码短，但细节容易在 `*` 的空判断、`#` 的复制范围、`%` 的反转对象上出错。

## 通过这道题可以学习到什么

这道题虽然代码不长，但非常适合练习“按规则模拟”的基本功。

### 1. 学会把题目规则翻译成代码

题目给了四种操作：

```text
小写字母：追加
*：删除末尾
#：复制当前结果
%：反转当前结果
```

代码中刚好对应四个分支：

```cpp
if (ch >= 'a' && ch <= 'z') {
    ans.push_back(ch);
} else if (ch == '*') {
    if (!ans.empty()) {
        ans.pop_back();
    }
} else if (ch == '#') {
    ans += ans;
} else if (ch == '%') {
    reverse(ans.begin(), ans.end());
}
```

这能训练你把自然语言规则转换成程序逻辑的能力。

以后遇到类似题目，比如“遇到某个符号执行某种操作”，都可以先列出规则表，再逐条写成 `if / else if`。

### 2. 学会使用一个变量维护当前状态

本题中最重要的变量是：

```cpp
string ans;
```

`ans` 表示当前已经处理出来的结果。

每处理一个字符 `ch`，`ans` 就更新一次。

这种思想叫做“维护状态”。

很多模拟题都会用到类似思想，例如：

- 当前分数 `score`
- 当前坐标 `(x, y)`
- 当前栈 `stk`
- 当前答案 `ans`
- 当前窗口内容 `window`

也就是说，这道题不仅是在练字符串，更是在练“状态如何随着操作一步步变化”。

### 3. 学会字符串的常见操作

这道题用到了 C++ `string` 的几个基础操作：

```cpp
ans.push_back(ch);
```

把一个字符追加到字符串末尾。

```cpp
ans.pop_back();
```

删除字符串最后一个字符。

```cpp
ans += ans;
```

把字符串自己复制一遍并拼接到末尾。

```cpp
reverse(ans.begin(), ans.end());
```

反转整个字符串。

这些都是 C++ 字符串题里非常常见的操作。

### 4. 学会处理边界情况

本题最典型的边界情况是：

```text
ans 为空时遇到 *
```

比如：

```text
s = "*a"
```

一开始：

```text
ans = ""
```

遇到 `*` 时不能直接删除最后一个字符，因为根本没有最后一个字符。

所以代码必须写：

```cpp
if (!ans.empty()) {
    ans.pop_back();
}
```

这能提醒你：凡是要删除、访问最后一个元素、访问数组下标时，都要先想清楚容器是否为空、下标是否越界。

### 5. 学会区分“当前结果”和“原始字符串”

本题里 `s` 是原始输入字符串，`ans` 是处理过程中的当前结果。

`%` 反转的是 `ans`，不是 `s`。

`#` 复制的也是 `ans`，不是 `s`。

这是很多人容易混淆的地方。

这道题可以帮你建立一个重要意识：

```text
输入数据不一定就是最终被操作的数据。
```

很多题都会有一个“原始输入”和一个“当前状态”。

你需要看清楚题目操作的是哪一个。

### 6. 学会用表格手动推演代码

题解里的例子都用到了类似表格：

| 轮次 | 当前字符 `ch` | 操作前 `ans` | 操作后 `ans` |
|---:|---|---|---|
| 1 | `'a'` | `""` | `"a"` |
| 2 | `'#'` | `"a"` | `"aa"` |

这种表格可以帮助你检查代码逻辑。

### 7. 学会分析复杂度

本题中：

- 遍历 `s` 是一层循环。

## 还可以延伸学习什么知识

这道题可以继续延伸到下面这些知识点。

### 1. 栈的思想

`*` 的行为是删除最后一个字符：

```cpp
ans.pop_back();
```

这和栈的 `pop` 很像。

栈的特点是：

```text
后进先出
```

也就是说，最后加入的元素最先被删除。

类似题目有：

- 删除字符串中的所有相邻重复项
- 有效的括号
- 简化路径
- 基本计算器类问题

### 2. 模拟题的通用写法

很多模拟题都可以按这个模板思考：

```text
1. 定义一个变量保存当前状态
2. 从左到右遍历输入
3. 根据当前字符或当前操作更新状态
4. 遍历结束后返回状态
```

本题中对应为：

```text
当前状态：ans
当前操作：ch
更新方式：追加、删除、复制、反转
最终返回：ans
```

掌握这个模板后，很多字符串模拟题都会更容易下手。

### 3. 原地操作和新建结果的区别

本题中我们没有修改原字符串 `s`，而是新建了一个结果字符串 `ans`。

这是一种很常见的写法。

有些题适合直接修改原数组或原字符串，有些题更适合维护一个新结果。

判断方法可以看：

- 原数据是否还需要继续读取？
- 修改原数据会不会影响后续判断？
- 是否需要返回一个全新的结果？

本题中 `s` 只是操作指令，真正变化的是 `ans`，所以新建 `ans` 最清晰。

### 4. C++ 标准库函数

这道题用到了：

```cpp
reverse(ans.begin(), ans.end());
```

这是 C++ 标准库里非常常用的函数。

你还可以继续学习这些常见函数：

```cpp
sort(v.begin(), v.end());
max(a, b);
min(a, b);
swap(a, b);
count(v.begin(), v.end(), x);
find(v.begin(), v.end(), x);
```

熟悉标准库可以让代码更短、更稳定，也能减少自己手写基础操作时出错。

### 5. 字符判断

本题用：

```cpp
ch >= 'a' && ch <= 'z'
```

判断一个字符是不是小写字母。

也可以学习 `<cctype>` 中的函数：

```cpp
islower(ch);
isalpha(ch);
isdigit(ch);
```

不过在 LeetCode 中，直接用范围判断也很常见，而且可读性很好。

### 6. 为进阶版本做准备

这道题是 I，通常 I 类题目数据规模比较小，可以直接模拟。

如果以后遇到进阶版本，可能会出现：

- 字符串非常长
- `#` 复制导致结果爆炸
- 不能真的构造完整字符串
- 只要求返回某个位置的字符

那时就可能需要学习：

- 逆向推导
- 长度计算
- 惰性处理
- 双端队列
- 栈
- 字符串哈希

所以本题是一个很好的基础入口：先把直接模拟写明白，再去理解更复杂的优化方法。

## 学完这题后的自查问题

你可以用下面几个问题检查自己是否真正掌握：

1. 如果 `ans` 为空时遇到 `*`，代码会发生什么？
2. `#` 复制的是原字符串 `s`，还是当前结果 `ans`？
3. `%` 反转的是已经处理过的部分，还是还没处理的部分？
4. 为什么 `ans += ans` 不是 `O(1)`？
5. 如果输入是 `"ab%c"`，最终结果是什么？
6. 如果输入是 `"a##"`，每一步的 `ans` 分别是什么？

参考答案：

```text
1. 什么也不做，因为代码会先判断 !ans.empty()
2. 复制当前结果 ans
3. 反转当前结果 ans
4. 因为需要复制当前 ans 中的所有字符
5. "bac"
6. "a" -> "aa" -> "aaaa"
```
