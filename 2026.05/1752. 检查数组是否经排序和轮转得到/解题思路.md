# 1752. 检查数组是否经排序和轮转得到 - 解题思路

## 题目描述

给你一个数组 `nums` 。`nums` 的源数组中，所有元素与 `nums` 相同，但按非递减顺序排列。

如果 `nums` 能够由源数组轮转若干位置（包括 0 个位置）得到，则返回 `true` ；否则，返回 `false` 。

源数组中可能存在 **重复项** 。

**注意：** 数组 `A` 在轮转 `x` 个位置后得到长度相同的数组 `B` ，使得对于每一个有效的下标 `i`，满足 `B[i] == A[(i+x) % A.length]` 。

**示例 1：**
```
输入：nums = [3,4,5,1,2]
输出：true
解释：[1,2,3,4,5] 为轮转后的源数组。
```

**示例 2：**
```
输入：nums = [2,1,3,4]
输出：false
解释：源数组无法经轮转得到 nums。
```

**示例 3：**
```
输入：nums = [1,2,3]
输出：true
解释：[1,2,3] 为轮转后的源数组。
```

---

## 题目理解

**什么是数组轮转？**
- 将数组前面的若干元素移到数组末尾
- 例如：`[1, 2, 3, 4, 5]` 轮转 2 次 → `[3, 4, 5, 1, 2]`

**目标：**
- 给定数组 `nums`，判断它是否可以由一个**非递减排序**的数组轮转得到

---

## 核心观察

### 观察 1：轮转后的数组特征

一个非递减数组轮转后，**最多只有一个下降点**（即 `nums[i] > nums[i+1]` 的位置）。

```
原数组（非递减）：[1, 2, 3, 4, 5]
轮转 2 次：      [3, 4, 5, 1, 2]
                      ↑     ↑
                   上升  下降点（只有 1 个）
```

### 观察 2：下降点数量判断

| 下降点数量 | 含义 | 结果 |
|-----------|------|------|
| 0 | 数组完全有序（轮转 0 次） | true |
| 1 | 可能是轮转后的结果 | true |
| ≥ 2 | 不可能是轮转后的结果 | false |

### 观察 3：环形比较

需要考虑数组的环形特性，即最后一个元素和第一个元素的比较。

```
nums = [3, 4, 5, 1, 2]

环形比较：
  3 vs 4 → 上升
  4 vs 5 → 上升
  5 vs 1 → 下降！（下降点）
  1 vs 2 → 上升
  2 vs 3 → 上升（环形：最后一个 vs 第一个）

下降点数量 = 1 → true
```

## 解法一：计数下降点（推荐）

### 核心思想

遍历数组，统计有多少个位置满足 `nums[i] > nums[(i+1) % n]`。如果数量 ≤ 1，返回 true。

### 图解过程

以 `nums = [3, 4, 5, 1, 2]` 为例：

```
i = 0: nums[0]=3 vs nums[1]=4 → 3 < 4，不是下降点
i = 1: nums[1]=4 vs nums[2]=5 → 4 < 5，不是下降点
i = 2: nums[2]=5 vs nums[3]=1 → 5 > 1，是下降点！dropCount = 1
i = 3: nums[3]=1 vs nums[4]=2 → 1 < 2，不是下降点
i = 4: nums[4]=2 vs nums[0]=3 → 2 < 3，不是下降点（环形比较）

dropCount = 1 ≤ 1 → return true
```

以 `nums = [2, 1, 3, 4]` 为例：

```
i = 0: nums[0]=2 vs nums[1]=1 → 2 > 1，是下降点！dropCount = 1
i = 1: nums[1]=1 vs nums[2]=3 → 1 < 3，不是下降点
i = 2: nums[2]=3 vs nums[3]=4 → 3 < 4，不是下降点
i = 3: nums[3]=4 vs nums[0]=2 → 4 > 2，是下降点！dropCount = 2

dropCount = 2 > 1 → return false
```

### 代码详解

```cpp
class Solution {
public:
    bool check(vector<int>& nums) {
        int n = nums.size();
        int dropCount = 0;  // 下降点计数

        // 遍历数组，统计下降点数量
        for (int i = 0; i < n; i++) {
            if (nums[i] > nums[(i + 1) % n]) {  // 环形比较
                dropCount++;
            }
        }

        // 下降点数量 <= 1 则为合法轮转
        return dropCount <= 1;
    }
};
```

### 关键点

- `(i + 1) % n` 实现环形比较：当 `i = n-1` 时，`(i+1) % n = 0`，即最后一个元素和第一个元素比较
- 为什么是 `<= 1` 而不是 `== 1`？因为轮转 0 次（原数组有序）也是合法的

### 复杂度分析

- **时间复杂度**：O(n)（一次遍历）
- **空间复杂度**：O(1)（只用常数空间）

## 解法二：模拟检查

### 核心思想

枚举所有可能的轮转起点，检查从该起点开始是否能得到非递减序列。

### 图解过程

以 `nums = [3, 4, 5, 1, 2]` 为例：

```
尝试 start = 0: 检查 [3, 4, 5, 1, 2]
  3 ≤ 4 ✓
  4 ≤ 5 ✓
  5 ≤ 1 ✗ → 失败

尝试 start = 1: 检查 [4, 5, 1, 2, 3]
  4 ≤ 5 ✓
  5 ≤ 1 ✗ → 失败

尝试 start = 2: 检查 [5, 1, 2, 3, 4]
  5 ≤ 1 ✗ → 失败

尝试 start = 3: 检查 [1, 2, 3, 4, 5]
  1 ≤ 2 ✓
  2 ≤ 3 ✓
  3 ≤ 4 ✓
  4 ≤ 5 ✓ → 成功！

return true
```

### 代码详解

```cpp
class Solution2 {
public:
    bool check(vector<int>& nums) {
        int n = nums.size();

        // 枚举每个可能的轮转起点
        for (int start = 0; start < n; start++) {
            bool valid = true;

            // 检查从 start 开始是否非递减
            for (int i = 0; i < n - 1; i++) {
                int curr = (start + i) % n;
                int next = (start + i + 1) % n;
                if (nums[curr] > nums[next]) {
                    valid = false;
                    break;
                }
            }

            if (valid) return true;
        }

        return false;
    }
};
```

### 复杂度分析

- **时间复杂度**：O(n²)（最坏情况枚举 n 个起点，每个检查 n 次）
- **空间复杂度**：O(1)

## 两种方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| 计数下降点 | O(n) | O(1) | 效率最高，代码简洁 | 需要理解环形比较 |
| 模拟检查 | O(n²) | O(1) | 直观易懂 | 效率较低 |

**推荐：** 方法一（计数下降点）效率最优，只需一次遍历。

## 常见错误

### 错误 1：忘记环形比较

```cpp
// 错误：只比较相邻元素，没有考虑首尾
if (nums[i] > nums[i + 1]) dropCount++;

// 正确：环形比较
if (nums[i] > nums[(i + 1) % n]) dropCount++;
```

### 错误 2：判断条件错误

```cpp
// 错误：只允许恰好 1 个下降点
return dropCount == 1;

// 正确：允许 0 个（原数组有序）或 1 个下降点
return dropCount <= 1;
```

### 错误 3：边界条件处理

```cpp
// 错误：空数组或单元素数组未处理
// 正确：n <= 1 时直接返回 true（空数组和单元素数组总是合法的）
if (n <= 1) return true;
```
