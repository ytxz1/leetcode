# 1871. 跳跃游戏 VII - 解题思路

## 题目描述

给你一个下标从 0 开始的二进制字符串 `s` 和两个整数 `minJump` 和 `maxJump`。

一开始，你在下标 0 处，且该位置的值一定为 `'0'`。

当同时满足如下条件时，你可以从下标 `i` 移动到下标 `j` 处：
- `i + minJump <= j <= min(i + maxJump, s.length - 1)` 且
- `s[j] == '0'`

如果你可以到达 `s` 的下标 `s.length - 1` 处，请你返回 `true`，否则返回 `false`。

**示例 1：**
```
输入：s = "011010", minJump = 2, maxJump = 3
输出：true
解释：从下标 0 出发，可以按 0 → 3 → 5 的路径到达下标 5。
```

**示例 2：**
```
输入：s = "01101110", minJump = 2, maxJump = 3
输出：false
```

---

## 题目理解

**跳跃规则：**
- 从位置 `i` 可以跳到位置 `j`，当且仅当：
  - 距离在 `[minJump, maxJump]` 范围内
  - 目标位置 `s[j] == '0'`

**目标：**
- 判断能否从位置 0 到达位置 `n-1`

---

## 核心思路

### 关键观察

对于位置 `j`，能跳到它的来源位置 `i` 必须满足：
- `i + minJump <= j <= i + maxJump`
- 即：`j - maxJump <= i <= j - minJump`

所以，`dp[j] = true` 当且仅当：
- `s[j] == '0'`
- 在区间 `[j - maxJump, j - minJump]` 内存在某个 `i` 使得 `dp[i] = true`

### 优化：前缀和

如果暴力检查每个区间，时间复杂度是 O(n × (maxJump - minJump))，可能超时。

**前缀和优化：**
- 定义 `prefix[i] = dp[0] + dp[1] + ... + dp[i-1]`
- 区间 `[left, right]` 内可达点的数量 = `prefix[right+1] - prefix[left]`
- 如果数量 > 0，说明该区间内有可达点

---

## 解法一：DP + 前缀和优化（推荐）

### 图解过程

以 `s = "011010"`, `minJump = 2`, `maxJump = 3` 为例：

```
下标:  0  1  2  3  4  5
字符:  0  1  1  0  1  0
dp:    T  F  F  ?  F  ?

Step 1: i = 0, dp[0] = true (起点)
  prefix = [0, 1, 1, 1, 1, 1, 1]

Step 2: i = 1, s[1] = '1' → dp[1] = false
  来源范围: [max(0, 1-3), 1-2] = [0, -1] → 无效
  prefix = [0, 1, 1, 1, 1, 1, 1]

Step 3: i = 2, s[2] = '1' → dp[2] = false
  来源范围: [max(0, 2-3), 2-2] = [0, 0]
  prefix[1] - prefix[0] = 1 > 0 → 但 s[2]='1'，所以 dp[2] = false
  prefix = [0, 1, 1, 1, 1, 1, 1]

Step 4: i = 3, s[3] = '0'
  来源范围: [max(0, 3-3), 3-2] = [0, 1]
  prefix[2] - prefix[0] = 1 > 0 → dp[3] = true ✓
  prefix = [0, 1, 1, 1, 2, 2, 2]

Step 5: i = 4, s[4] = '1' → dp[4] = false
  prefix = [0, 1, 1, 1, 2, 2, 2]

Step 6: i = 5, s[5] = '0'
  来源范围: [max(0, 5-3), 5-2] = [2, 3]
  prefix[4] - prefix[2] = 2 - 1 = 1 > 0 → dp[5] = true ✓

最终: dp[5] = true → return true
```

### 代码详解

```cpp
class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n = s.size();
        if (s[n - 1] == '1') return false;

        vector<bool> dp(n, false);
        dp[0] = true;

        // prefix[i] = dp[0] + dp[1] + ... + dp[i-1]
        vector<int> prefix(n + 1, 0);
        prefix[1] = 1;  // dp[0] = true

        for (int i = 1; i < n; i++) {
            if (s[i] == '0') {
                // 能到达 i 的来源范围：[i-maxJump, i-minJump]
                int left = max(0, i - maxJump);
                int right = i - minJump;

                if (right >= 0) {
                    // 用前缀和判断 [left, right] 范围内是否有可达点
                    int count = prefix[right + 1] - prefix[left];
                    dp[i] = (count > 0);
                }
            }
            // 更新前缀和
            prefix[i + 1] = prefix[i] + (dp[i] ? 1 : 0);
        }

        return dp[n - 1];
    }
};
```

### 关键点

1. **来源范围计算**：对于位置 `i`，能跳到它的位置范围是 `[i-maxJump, i-minJump]`
2. **前缀和查询**：`prefix[right+1] - prefix[left]` 快速计算区间内可达点数量
3. **边界处理**：`right >= 0` 才需要检查，否则没有来源

### prefix 数组详解

**定义：**
- `prefix[i] = dp[0] + dp[1] + ... + dp[i-1]`
- 即前 `i` 个位置中可达点的数量

**作用：** 快速判断一个区间内是否存在可达点

**查询公式：**
```
区间 [left, right] 内可达点的数量 = prefix[right + 1] - prefix[left]
```

**具体例子：**

以 `s = "011010"`, `minJump = 2`, `maxJump = 3` 为例：

```
下标:  0  1  2  3  4  5
字符:  0  1  1  0  1  0
dp:    T  F  F  T  F  T

计算 dp[3] 时：
  来源范围: [3-3, 3-2] = [0, 1]
  
  不用 prefix：需要遍历 [0,1]，检查 dp[0] 和 dp[1]，O(区间长度)
  用 prefix：prefix[2] - prefix[0] = 1 - 0 = 1 > 0 → 有可达点，O(1)

prefix 数组构建过程：
  prefix[0] = 0
  prefix[1] = dp[0] = 1
  prefix[2] = dp[0]+dp[1] = 1+0 = 1
  prefix[3] = dp[0]+dp[1]+dp[2] = 1+0+0 = 1
  prefix[4] = dp[0]+dp[1]+dp[2]+dp[3] = 1+0+0+1 = 2
  prefix[5] = 1+0+0+1+0 = 2
  prefix[6] = 1+0+0+1+0+1 = 3
```

**为什么需要 prefix？**

| 方式 | 每个位置的查询时间 | 总时间复杂度 |
|------|------------------|-------------|
| 暴力遍历区间 | O(maxJump - minJump) | O(n × (maxJump - minJump))，可能超时 |
| 前缀和 | O(1) | O(n)，最优 |

简单说，`prefix` 把"区间求和"从 O(区间长度) 优化到了 O(1)。

### 复杂度分析

- **时间复杂度**：O(n)（一次遍历，前缀和查询 O(1)）
- **空间复杂度**：O(n)（dp 数组 + 前缀和数组）

## 解法二：BFS + 滑动窗口优化

### 核心思想

用 BFS 遍历所有可达位置，用变量 `maxReach` 记录已处理的最大位置，避免重复遍历。

### 图解过程

以 `s = "011010"`, `minJump = 2`, `maxJump = 3` 为例：

```
初始: q = [0], maxReach = 0

Step 1: i = 0
  可跳范围: [0+2, 0+3] = [2, 3]
  实际范围: [max(2, 0+1), min(3, 5)] = [2, 3]
  检查: s[2]='1'(跳过), s[3]='0'(加入队列)
  q = [3], maxReach = 3

Step 2: i = 3
  可跳范围: [3+2, 3+3] = [5, 6]
  实际范围: [max(5, 3+1), min(6, 5)] = [5, 5]
  检查: s[5]='0'(加入队列, 且 j=5=n-1, 返回 true)
  q = [5], maxReach = 5

return true
```

### 代码详解

```cpp
class Solution2 {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n = s.size();
        if (s[n - 1] == '1') return false;

        queue<int> q;
        q.push(0);

        int maxReach = 0;  // 记录已处理的最大位置

        while (!q.empty()) {
            int i = q.front();
            q.pop();

            // 从 i 能跳到的范围：[i+minJump, i+maxJump]
            int start = max(i + minJump, maxReach + 1);
            int end = min(i + maxJump, n - 1);

            for (int j = start; j <= end; j++) {
                if (s[j] == '0') {
                    if (j == n - 1) return true;
                    q.push(j);
                }
            }

            maxReach = end;  // 更新已处理的最大位置
        }

        return false;
    }
};
```

### 关键点

- `maxReach` 的作用：避免重复遍历已处理过的位置
- `start = max(i + minJump, maxReach + 1)`：只处理未遍历过的位置

### 复杂度分析

- **时间复杂度**：O(n)（每个位置最多被访问一次）
- **空间复杂度**：O(n)（队列空间）

## 两种方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| DP + 前缀和 | O(n) | O(n) | 效率最优，代码简洁 | 需要理解前缀和 |
| BFS + 滑动窗口 | O(n) | O(n) | 直观易懂 | 代码稍复杂 |

**推荐：** 两种方法都是 O(n)，DP 方法更简洁。

## 常见错误

### 错误 1：来源范围计算错误

```cpp
// 错误：把跳跃方向搞反
int left = i + minJump;
int right = i + maxJump;

// 正确：来源范围是 [i-maxJump, i-minJump]
int left = max(0, i - maxJump);
int right = i - minJump;
```

### 错误 2：忘记检查目标位置

```cpp
// 错误：只检查范围，不检查 s[i] 是否为 '0'
if (count > 0) dp[i] = true;

// 正确：必须先检查 s[i] == '0'
if (s[i] == '0' && count > 0) dp[i] = true;
```

### 错误 3：前缀和索引错误

```cpp
// 错误：索引越界
int count = prefix[right] - prefix[left - 1];

// 正确：注意 +1 偏移
int count = prefix[right + 1] - prefix[left];
```

### 错误 4：BFS 中忘记更新 maxReach

```cpp
// 错误：没有 maxReach，导致重复遍历
for (int j = i + minJump; j <= end; j++) { ... }

// 正确：用 maxReach 避免重复
int start = max(i + minJump, maxReach + 1);
```
