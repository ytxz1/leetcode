# 3614. 用特殊操作处理字符串 II

题目链接：[LeetCode 3614. 用特殊操作处理字符串 II](https://leetcode.cn/problems/process-string-with-special-operations-ii/)

## 题目描述

给你一个字符串 `s` 和一个整数 `k`。

字符串 `s` 中包含小写英文字母，以及三个特殊字符：

```text
*
#
%
```

从左到右处理 `s`，得到一个结果字符串 `ans`。

处理规则如下：

| 当前字符 `ch` | 操作 |
|---|---|
| 小写字母 | 把 `ch` 追加到 `ans` 的末尾 |
| `*` | 删除 `ans` 的最后一个字符；如果 `ans` 为空，则什么也不做 |
| `#` | 把当前 `ans` 复制一份，并追加到 `ans` 后面 |
| `%` | 反转当前 `ans` |

处理完整个字符串后，返回最终字符串中下标为 `k` 的字符。

如果 `k` 超出最终字符串的范围，返回：

```text
'.'
```

这里的 `k` 是从 `0` 开始的下标。

## 和 3612 题的区别

3612. 用特殊操作处理字符串 I 要求返回完整的最终字符串。

如果直接模拟，可以写：

```cpp
string ans;
for (char ch : s) {
    ...
}
return ans;
```

但 3614 是 II，重点变了：它只问最终字符串的第 `k` 个字符。

因为 `#` 会复制当前字符串，字符串长度可能增长得非常快。

例如：

```text
s = "a##########"
```

每遇到一个 `#`，当前字符串长度就翻倍。

长度变化：

```text
1 -> 2 -> 4 -> 8 -> 16 -> 32 -> ...
```

如果特殊字符很多，最终字符串可能非常长，不能真的构造完整 `ans`。

所以本题的核心不是“构造字符串”，而是：

```text
只追踪长度，再倒着推回 k 对应的原始字符。
```

## 核心思路

分两步。

第一步：正向扫描 `s`，只计算每一步处理后的字符串长度。

代码变量：

```cpp
vector<long long> lens(n);
long long len = 0;
```

含义：

| 变量 | 含义 |
|---|---|
| `len` | 当前处理后的字符串长度 |
| `lens[i]` | 处理完 `s[i]` 后，当前字符串的长度 |

第二步：如果 `k` 在最终长度范围内，就从右往左扫描 `s`，把当前下标 `k` 反向映射回前一步。

逆向扫描时的变量：

```cpp
long long curLen = lens[i];
long long prevLen = (i == 0 ? 0 : lens[i - 1]);
```

含义：

| 变量 | 含义 |
|---|---|
| `i` | 当前正在逆向处理的位置 |
| `ch` | 当前字符，也就是 `s[i]` |
| `curLen` | 处理完 `s[i]` 后的长度 |
| `prevLen` | 处理 `s[i]` 之前的长度 |
| `k` | 目标字符在当前这一步结果字符串中的下标 |

我们倒着看每一种操作。

## 逆向还原规则

### 1. 当前字符是小写字母

正向处理时：

```cpp
ans.push_back(ch);
```

也就是把 `ch` 追加到末尾。

长度变化：

```text
prevLen -> prevLen + 1
```

如果逆向时：

```text
k == curLen - 1
```

说明目标位置正好是这个刚追加的字符。

此时答案就是：

```cpp
return ch;
```

如果 `k` 不是最后一个位置，说明目标字符来自更早的字符串，继续往前推即可。

### 2. 当前字符是 `*`

正向处理时：

```cpp
if (!ans.empty()) {
    ans.pop_back();
}
```

它只会删除末尾字符。

对于最终还保留下来的位置来说，下标不变。

例如删除前：

```text
abc
```

删除后：

```text
ab
```

删除后的下标：

```text
0 -> a
1 -> b
```

和删除前的下标仍然一致。

所以逆向遇到 `*` 时，`k` 不需要变化。

### 3. 当前字符是 `#`

正向处理时：

```cpp
ans += ans;
```

假设操作前：

```text
ans = "abc"
```

操作后：

```text
ans = "abcabc"
```

长度变化：

```text
prevLen -> 2 * prevLen
```

逆向时有两种情况。

如果：

```text
k < prevLen
```

说明目标位置在第一份里，下标不变。

如果：

```text
k >= prevLen
```

说明目标位置在第二份里，要映射回第一份：

```cpp
k -= prevLen;
```

比如：

```text
操作前 ans = "abc"，prevLen = 3
操作后 ans = "abcabc"
```

下标 `4` 在第二份中，对应第一份下标：

```text
4 - 3 = 1
```

也就是字符 `'b'`。

### 4. 当前字符是 `%`

正向处理时：

```cpp
reverse(ans.begin(), ans.end());
```

假设操作前：

```text
ans = "abcd"
```

操作后：

```text
ans = "dcba"
```

长度不变。

逆向时，当前下标 `k` 要映射回反转前的位置：

```cpp
k = prevLen - 1 - k;
```

例如当前反转后的字符串是：

```text
d c b a
0 1 2 3
```

如果 `k = 1`，当前字符是 `'c'`。

它在反转前 `"abcd"` 中的下标是：

```text
4 - 1 - 1 = 2
```

也就是 `'c'`。

## 代码实现

核心代码见 [solution.cpp](D:\leetcode\2026.6\3614. 用特殊操作处理字符串 II\solution.cpp)。

LeetCode 提交时只需要提交下面这个 `Solution` 类：

```cpp
class Solution {
public:
    char processStr(string s, long long k) {
        const long long INF = (1LL << 62);
        int n = static_cast<int>(s.size());
        vector<long long> lens(n);

        long long len = 0;
        for (int i = 0; i < n; i++) {
            char ch = s[i];

            if (ch >= 'a' && ch <= 'z') {
                len = min(INF, len + 1);
            } else if (ch == '*') {
                if (len > 0) {
                    len--;
                }
            } else if (ch == '#') {
                len = min(INF, len * 2);
            } else if (ch == '%') {
                // Reversing does not change the length.
            }

            lens[i] = len;
        }

        if (k < 0 || k >= len) {
            return '.';
        }

        for (int i = n - 1; i >= 0; i--) {
            char ch = s[i];
            long long curLen = lens[i];
            long long prevLen = (i == 0 ? 0 : lens[i - 1]);

            if (ch >= 'a' && ch <= 'z') {
                if (k == curLen - 1) {
                    return ch;
                }
            } else if (ch == '#') {
                if (prevLen > 0 && k >= prevLen) {
                    k -= prevLen;
                }
            } else if (ch == '%') {
                k = prevLen - 1 - k;
            }
        }

        return '.';
    }
};
```

## 为什么要限制最大长度 `INF`

代码中写了：

```cpp
const long long INF = (1LL << 62);
```

并且更新长度时写：

```cpp
len = min(INF, len * 2);
```

原因是 `#` 会让长度翻倍，极端情况下长度可能超过 `long long` 的安全范围。

但是我们不需要知道特别巨大的精确长度。

只要长度已经大到足够判断 `k` 是否在范围内，就可以截断到一个很大的值 `INF`。

这个技巧叫做：

```text
长度截断
```

它常用于“只关心第 k 个字符”或“只关心是否超过某个范围”的题目。

## 示例 1：`s = "a#b%*"`, `k = 0`

先直接模拟一下完整结果，帮助理解：

```text
初始 ans = ""

'a' -> "a"
'#' -> "aa"
'b' -> "aab"
'%' -> "baa"
'*' -> "ba"
```

最终字符串是：

```text
"ba"
```

下标：

```text
0   1
b   a
```

所以 `k = 0` 时答案是：

```text
'b'
```

### 正向计算长度

代码变量：

```cpp
len = 0
lens = []
```

逐步过程：

| i | ch | 操作 | 操作前 `len` | 操作后 `len` | `lens[i]` |
|---:|---|---|---:|---:|---:|
| 0 | `'a'` | 追加一个字符 | 0 | 1 | 1 |
| 1 | `'#'` | 长度翻倍 | 1 | 2 | 2 |
| 2 | `'b'` | 追加一个字符 | 2 | 3 | 3 |
| 3 | `'%'` | 反转，长度不变 | 3 | 3 | 3 |
| 4 | `'*'` | 删除末尾 | 3 | 2 | 2 |

最终：

```text
len = 2
lens = [1, 2, 3, 3, 2]
```

因为：

```text
k = 0 < len = 2
```

所以答案存在，开始逆向寻找。

### 逆向还原 `k`

初始：

```text
k = 0
```

逆向过程：

| i | ch | `curLen` | `prevLen` | 操作前 `k` | 逆向规则 | 操作后 `k` |
|---:|---|---:|---:|---:|---|---:|
| 4 | `'*'` | 2 | 3 | 0 | 删除末尾，保留位置不变 | 0 |
| 3 | `'%'` | 3 | 3 | 0 | 反转映射：`k = 3 - 1 - 0` | 2 |
| 2 | `'b'` | 3 | 2 | 2 | `k == curLen - 1`，命中字母 `'b'` | 返回 |

最后返回：

```text
'b'
```

这个过程和完整字符串 `"ba"` 的下标 `0` 对应一致。

## 正确性说明

我们证明算法返回的字符就是最终字符串中下标为 `k` 的字符。

第一步，正向扫描时，`lens[i]` 记录处理完 `s[i]` 后的字符串长度。

对于每个字符：

- 小写字母会让长度加 `1`
- `*` 在长度大于 `0` 时让长度减 `1`
- `#` 让长度翻倍
- `%` 不改变长度

这和题目规则完全一致，所以 `lens[i]` 记录的长度正确。

第二步，如果：

```text
k >= 最终长度
```

说明最终字符串中不存在下标 `k`，返回 `'.'` 正确。

第三步，逆向扫描时，假设当前 `k` 表示目标字符在“处理完 `s[i]` 后的字符串”中的位置。

我们根据 `s[i]` 的操作，把它映射回“处理 `s[i]` 之前的字符串”中的位置。

- 如果 `s[i]` 是小写字母，并且 `k` 是最后一个位置，那么目标字符就是这个字母。
- 如果 `s[i]` 是小写字母，但 `k` 不是最后一个位置，那么目标字符来自更早之前，位置不变。
- 如果 `s[i]` 是 `*`，删除的是末尾字符，保留下来的字符下标不变。
- 如果 `s[i]` 是 `#`，后半段是前半段的复制，所以后半段位置 `k` 映射为 `k - prevLen`。
- 如果 `s[i]` 是 `%`，反转前后的下标关系是 `k = prevLen - 1 - k`。

因此每一步逆向映射都保持“当前 `k` 指向的字符不变”。

当逆向遇到某个小写字母，并且 `k` 正好落在它被追加的位置时，这个字母就是最终答案。

所以算法正确。

## 复杂度分析

设 `n` 是字符串 `s` 的长度。

时间复杂度：

```text
O(n)
```

原因：

- 正向计算长度扫描一次 `s`
- 逆向还原 `k` 最多再扫描一次 `s`

空间复杂度：

```text
O(n)
```

原因是使用了数组：

```cpp
vector<long long> lens(n);
```

保存每一步处理后的长度。

## 常见错误

### 错误 1：直接构造完整字符串

如果写：

```cpp
ans += ans;
```

在 II 中可能导致字符串非常大，内存超限或运行超时。

本题应该只记录长度，并逆向还原位置。

### 错误 2：忘记 `k` 是从 0 开始

如果最终字符串是：

```text
"ba"
```

那么：

```text
k = 0 -> 'b'
k = 1 -> 'a'
```

不是从 `1` 开始数。

### 错误 3：反转映射公式写错

对于 `%`，正确写法是：

```cpp
k = prevLen - 1 - k;
```

不是：

```cpp
k = curLen - k;
```

因为下标从 `0` 开始，所以必须减 `1`。

### 错误 4：遇到 `#` 时没有判断第二份

对于 `#`：

```text
当前串 = 前一串 + 前一串
```

如果 `k` 在第二份：

```cpp
k -= prevLen;
```

如果 `k` 在第一份，`k` 不变。

### 错误 5：长度翻倍溢出

如果不断遇到 `#`，长度可能非常大。

所以代码用：

```cpp
len = min(INF, len * 2);
```

避免 `long long` 溢出。

## 通过这道题可以学习到什么

这道题比 3612 更进了一步，很适合学习“不能直接模拟时怎么办”。

### 1. 学会从“构造答案”转为“定位答案”

3612 可以直接构造完整字符串。

但 3614 只问第 `k` 个字符。

这时没有必要，也不应该构造完整字符串。

更好的思路是：

```text
我不需要知道整个字符串是什么。
我只需要知道第 k 个字符来自哪里。
```

这是一种非常重要的算法思维。

很多题都可以从“求完整结果”优化成“只求目标位置”。

### 2. 学会正向计算信息，逆向还原答案

本题的结构是：

```text
正向：计算每一步的长度
逆向：根据操作反推出 k 的来源
```

这类思路在算法题中很常见。

例如：

- 先计算总长度，再从后往前找第 `k` 个字符
- 先执行所有操作的统计结果，再倒着撤销操作
- 先知道最终状态，再反推初始来源

这种能力很有用，因为有些问题正着做很难，倒着做反而清楚。

### 3. 学会下标映射

本题最关键的是把 `k` 从当前字符串映射回前一个字符串。

例如：

```text
复制：abc -> abcabc
```

如果 `k = 4`，它在第二份中。

映射回原来的 `"abc"`：

```text
k = 4 - 3 = 1
```

再比如：

```text
反转：abcd -> dcba
```

如果反转后 `k = 1`，映射回反转前：

```text
k = 4 - 1 - 1 = 2
```

这种“下标映射”是很多进阶题的核心。

### 4. 学会处理超大长度

本题中最终字符串可能非常长。

我们使用：

```cpp
const long long INF = (1LL << 62);
len = min(INF, len * 2);
```

这可以防止溢出。

你可以学到一个技巧：

```text
当精确值过大但不需要精确知道时，可以把它截断到一个足够大的上限。
```

这个技巧常见于：

- 第 `k` 个字符
- 第 `k` 个排列
- 大数计数
- 动态规划计数超过上限

### 5. 学会区分“长度变化”和“内容变化”

不同操作对长度和内容的影响不同：

| 操作 | 内容是否变化 | 长度是否变化 |
|---|---|---|
| 小写字母 | 变化，末尾追加 | 加 1 |
| `*` | 变化，删除末尾 | 可能减 1 |
| `#` | 变化，复制一份 | 翻倍 |
| `%` | 变化，反转顺序 | 不变 |

这道题能训练你把操作拆成两个维度：

```text
长度怎么变？
k 的位置怎么变？
```

### 6. 学会写可验证的推演表

这份题解中用了很多表：

```text
i, ch, len, lens[i]
i, ch, curLen, prevLen, k
```

写表不是为了形式好看，而是为了帮助你调试。

当代码错了，你可以问自己：

```text
这一轮 len 算对了吗？
这一轮 prevLen 是多少？
k 是落在第一份还是第二份？
反转后 k 映射对了吗？
```

这种调试方式对链表、数组、字符串、动态规划都很有帮助。

## 还可以延伸学习什么知识

### 1. 逆向思维

本题最重要的延伸知识就是逆向思维。

有些题正向模拟会爆炸，但逆向只需要追踪一个位置。

类似思想可以用于：

- 从最终状态反推初始状态
- 从目标位置反推来源位置
- 从结果倒推最后一步操作

### 2. 惰性计算

本题没有真正生成最终字符串，而是只维护长度和位置。

这是一种“惰性计算”的思想：

```text
能不算的东西就不算。
只计算回答问题所必需的信息。
```

这在大数据量问题中非常重要。

### 3. 字符串倍增问题

`#` 会让字符串倍增。

倍增问题经常不能直接展开，需要考虑：

- 长度变化
- 区间来源
- 下标映射

类似问题包括：

- 解码字符串后索引
- 第 k 个语法符号
- 无限重复字符串中的第 k 个字符

### 4. 溢出保护

当题目中出现：

```text
翻倍
指数增长
k 很大
```

就要警惕整数溢出。

常见保护方式：

```cpp
const long long INF = (1LL << 62);
value = min(INF, value * 2);
```

或者在乘法前判断：

```cpp
if (value > INF / 2) {
    value = INF;
} else {
    value *= 2;
}
```

### 5. 空间优化

本解法使用了：

```cpp
vector<long long> lens(n);
```

空间复杂度是 `O(n)`。

如果想进一步优化，可以思考：

- 是否可以只保存关键位置？
- 是否能重新正向计算某个前缀长度？
- 时间和空间之间如何取舍？

不过对于本题，`O(n)` 空间已经很清晰、稳妥。

### 6. 和栈思想的联系

`*` 删除末尾字符，很像栈的 `pop` 操作。

虽然本题最终没有真正使用栈，但理解 `*` 时可以借助栈：

```text
追加字符 -> push
删除末尾 -> pop
```

这能帮助你把它和很多经典题联系起来，比如：

- 有效的括号
- 删除字符串中的相邻重复项
- 简化路径

## 学完这题后的自查问题

你可以用下面的问题检查自己是否掌握。

1. 为什么本题不能直接构造最终字符串？
2. `lens[i]` 表示什么？
3. 遇到 `#` 时，如果 `k >= prevLen`，为什么要执行 `k -= prevLen`？
4. 遇到 `%` 时，为什么公式是 `k = prevLen - 1 - k`？
5. 遇到 `*` 时，为什么逆向过程中 `k` 不需要变？
6. 为什么要用 `INF` 截断长度？
7. 如果 `s = "a##"`，`k = 2`，答案是什么？

参考答案：

```text
1. 因为 # 会让字符串长度快速翻倍，完整构造可能超时或超内存。
2. lens[i] 表示处理完 s[i] 后的字符串长度。
3. 因为 k 落在复制出来的第二份中，要映射回第一份的相同位置。
4. 因为反转后下标 k 对应反转前的下标 prevLen - 1 - k。
5. 因为 * 删除的是末尾字符，保留下来的字符下标不变。
6. 因为长度可能超过 long long，截断可以防止溢出。
7. "a##" 最终是 "aaaa"，k = 2 的答案是 'a'。
```
