# 3558. 给边赋权值的方案数 I

## 题目理解

给你一棵无向树，节点编号为 `1` 到 `n`，其中 `1` 是根节点。

每条边可以赋值为 `1` 或 `2`。我们只需要考虑从根节点 `1` 到某个最深节点的路径，要求这条路径上的边权和为奇数，返回满足要求的赋权方案数。

答案需要对 `1e9 + 7` 取模。

本题真正影响答案的变量只有一个：

```text
maxDepth = 根节点 1 到最深节点的边数
```

注意：`maxDepth` 是边数，不是节点数。

## 核心变量

| 变量名 | 含义 |
|---|---|
| `edges` | 输入的无向边数组 |
| `n` | 节点数量，`n = edges.size() + 1` |
| `graph` | 邻接表，`graph[u]` 存储节点 `u` 的所有相邻节点 |
| `node` | 当前正在遍历的节点 |
| `parent` | 当前节点的父节点，用来防止无向树中走回头路 |
| `depth` | 当前节点到根节点 `1` 的边数 |
| `maxDepth` | 遍历过程中发现的最大深度 |
| `MOD` | 取模常数 `1000000007` |
| `base` | 快速幂中的底数，初始为 `2` |
| `exp` | 快速幂中的指数，初始为 `maxDepth - 1` |
| `ans` | 快速幂中当前累计的答案 |

## 核心结论

设从根节点 `1` 到最深节点的路径长度为 `d`，也就是路径上有 `d` 条边。

每条边有两种选择：

```text
1 或 2
```

如果一条边赋值为 `1`，它会改变路径和的奇偶性。

如果一条边赋值为 `2`，它不会改变路径和的奇偶性。

所以：

```text
路径和为奇数
= 路径上赋值为 1 的边数为奇数
```

长度为 `d` 的路径共有：

```text
2^d
```

种赋权方案，其中奇数和、偶数和各占一半，因此：

```text
answer = 2^(d - 1)
```

如果 `d = 0`，说明树中只有根节点，没有边，路径和是 `0`，不是奇数，答案是 `0`。

## 为什么答案是 2^(d - 1)

假设路径有 `d` 条边，分别记作：

```text
e1, e2, e3, ..., ed
```

固定前 `d - 1` 条边：

```text
e1, e2, ..., e(d-1)
```

此时前缀和只有两种情况：

| 前 `d - 1` 条边的和 | 最后一条边 `ed` 应该选什么 | 最终路径和 |
|---|---|---|
| 偶数 | `1` | 奇数 |
| 奇数 | `2` | 奇数 |

也就是说，前 `d - 1` 条边怎么选都可以，最后一条边被唯一确定。

前 `d - 1` 条边共有：

```text
2^(d - 1)
```

种选择，所以合法方案数就是：

```text
2^(d - 1)
```

## 详细例子一：只有一条边

### 输入

```text
edges = [[1, 2]]
```

### 第 1 步：计算节点数

```text
edges.size() = 1
n = edges.size() + 1 = 2
```

### 第 2 步：建图

初始：

```text
graph[1] = []
graph[2] = []
```

处理边 `[1, 2]`：

```text
u = 1
v = 2
graph[1].push_back(2)
graph[2].push_back(1)
```

建图完成：

```text
graph[1] = [2]
graph[2] = [1]
```

树结构：

```text
1 - 2
```

### 第 3 步：DFS 求最大深度

初始变量：

```text
maxDepth = 0
```

调用：

```text
dfs(node = 1, parent = 0, depth = 0)
```

步骤表：

| 步骤 | 当前调用 | `node` | `parent` | `depth` | `maxDepth` 更新 | 下一步 |
|---|---|---:|---:|---:|---:|---|
| 1 | `dfs(1, 0, 0)` | 1 | 0 | 0 | `max(0,0)=0` | 遍历 `graph[1]=[2]` |
| 2 | 访问邻居 `2` | 2 | - | - | - | `2 != parent(0)`，递归 |
| 3 | `dfs(2, 1, 1)` | 2 | 1 | 1 | `max(0,1)=1` | 遍历 `graph[2]=[1]` |
| 4 | 访问邻居 `1` | 1 | - | - | - | `1 == parent(1)`，跳过 |
| 5 | DFS 结束 | - | - | - | `maxDepth = 1` | 得到最大深度 |

最终：

```text
maxDepth = 1
d = 1
```

### 第 4 步：枚举赋权

路径是：

```text
1 -> 2
```

路径上只有一条边：

```text
e1
```

赋权枚举：

| `e1` | 路径和 `sum` | 是否奇数 | 是否合法 |
|---:|---:|---|---|
| 1 | 1 | 是 | 是 |
| 2 | 2 | 否 | 否 |

答案：

```text
2^(maxDepth - 1)
= 2^(1 - 1)
= 2^0
= 1
```

## 方法一：DFS 代码逻辑

核心代码：

```cpp
auto dfs = [&](auto&& self, int node, int parent, int depth) -> void {
    maxDepth = max(maxDepth, depth);

    for (int next : graph[node]) {
        if (next == parent) {
            continue;
        }
        self(self, next, node, depth + 1);
    }
};
```

每个变量的含义：

| 变量名 | 含义 |
|---|---|
| `self` | lambda 自己，用来实现递归调用 |
| `node` | 当前所在节点 |
| `parent` | 当前节点的父节点 |
| `depth` | 当前节点到根节点的边数 |
| `next` | 当前节点的一个相邻节点 |
| `maxDepth` | 目前见过的最大深度 |

执行逻辑：

1. 进入某个节点时，先用 `depth` 更新 `maxDepth`。
2. 遍历 `graph[node]`，找到所有相邻节点。
3. 如果相邻节点等于 `parent`，说明会走回父节点，必须跳过。
4. 否则递归访问 `next`，并把深度变成 `depth + 1`。

## 方法二：BFS 代码逻辑

核心代码：

```cpp
queue<pair<int, int>> q;
q.push({1, 0});

while (!q.empty()) {
    auto [node, depth] = q.front();
    q.pop();

    maxDepth = max(maxDepth, depth);

    for (int next : graph[node]) {
        if (visited[next]) {
            continue;
        }
        visited[next] = 1;
        q.push({next, depth + 1});
    }
}
```

队列中每个元素是：

```text
(node, depth)
```

也就是：

```text
当前节点编号 + 当前节点到根节点的边数
```

BFS 的特点是按层处理：

```text
深度 0：节点 1
深度 1：节点 2、3
深度 2：节点 4、5
```

因此 BFS 很适合求最大深度。

## 方法三：栈模拟 DFS 代码逻辑

核心代码：

```cpp
vector<int> parent(n + 1, 0);
vector<int> depth(n + 1, 0);
stack<int> st;
st.push(1);
parent[1] = -1;
```

变量含义：

| 变量名 | 含义 |
|---|---|
| `parent[x]` | 节点 `x` 的父节点 |
| `depth[x]` | 节点 `x` 到根节点 `1` 的边数 |
| `st` | 手动维护的 DFS 栈 |

当从 `node` 访问到 `next` 时：

```cpp
parent[next] = node;
depth[next] = depth[node] + 1;
st.push(next);
```

含义是：

1. `next` 是从 `node` 走过去的，所以 `node` 是 `next` 的父节点。
2. `next` 比 `node` 多走一条边，所以深度加 1。
3. 把 `next` 放入栈中，后面继续处理它的邻居。

## 易错点

### 易错点 1：把深度当成节点数

路径：

```text
1 -> 3 -> 4
```

节点数是 `3`，边数是 `2`。

本题给边赋值，所以必须用边数：

```text
depth = 2
```

### 易错点 2：统计最深节点数量

例子二中最深节点有两个：

```text
4 和 5
```

但是答案不是：

```text
最深节点数量 * 2^(d - 1)
```

本题只需要根据最大深度 `d` 计算方案数。

### 易错点 3：真的枚举所有赋权

如果：

```text
d = 100000
```

那么所有赋权有：

```text
2^100000
```

种，完全不能枚举。

正确做法是：

```text
answer = 2^(d - 1) % MOD
```

并使用快速幂。

### 易错点 4：忘记 `maxDepth = 0` 的情况

如果树只有一个节点：

```text
n = 1
edges = []
```

那么：

```text
maxDepth = 0
```

没有任何边可以赋值，路径和是 `0`，不是奇数。

答案应该是：

```text
0
```

## 总结

本题可以拆成两个问题：

1. 用 DFS / BFS / 栈模拟 DFS 求出 `maxDepth`。
2. 用快速幂计算 `2^(maxDepth - 1) % MOD`。

最终公式：

```text
if maxDepth == 0:
    answer = 0
else:
    answer = 2^(maxDepth - 1) % 1000000007
```
