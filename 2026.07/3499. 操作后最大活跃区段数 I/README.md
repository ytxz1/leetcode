# 3499. 操作后最大活跃区段数 I

## 题目理解

给你一个长度为 `n` 的二进制字符串 `s`：

- `1` 表示活跃区段；
- `0` 表示非活跃区段。

你最多可以执行一次 `trade` 操作，使最终 `s` 中的 `1` 的数量最大。

一次 `trade` 分两步：

1. 选择一段连续的 `1`，并且这段 `1` 左右都被 `0` 包围，把它们全部变成 `0`。
2. 然后选择一段连续的 `0`，并且这段 `0` 左右都被 `1` 包围，把它们全部变成 `1`。

注意：判断包围关系时，把字符串看成：

```text
t = "1" + s + "1"
```

但是两端额外补上的 `1` 不计入最终答案。

目标：返回执行最优操作后，原字符串 `s` 中最多有多少个 `1`。

## 关键结论

假设原字符串中有这样一段结构：

```text
左边 0 段      中间 1 段      右边 0 段
000...000      111...111      000...000
长度为 a       长度为 b       长度为 c
```

也就是：

```text
0^a 1^b 0^c
```

第一步把中间的 `1^b` 变成 `0`：

```text
0^a 1^b 0^c  ->  0^(a + b + c)
```

这会让这三段合并成一个更长的 `0` 段。

第二步再把这整个 `0` 段变成 `1`：

```text
0^(a + b + c) -> 1^(a + b + c)
```

最终变化是：

- 原来的中间 `b` 个 `1` 先丢失，后又变回来；
- 真正新增的是左右两边的 `a + c` 个 `0`。

所以：

```text
最终答案 = 原本 1 的数量 + max(相邻两个 0 段长度之和)
```

这里的「相邻两个 0 段」指它们中间隔着一段 `1`。

## 图解一：为什么只看两个 0 段

以 `s = "1000100"` 为例。

补上两端的 `1`：

```text
s =      1 000 1 00
t =    1 1 000 1 00 1
```

按连续段分组：

```text
t:  11 | 000 | 1 | 00 | 1
         a=3  b=1 c=2
```

选择中间这一段 `1`：

```text
第一步：把中间 1 变成 0

11 | 000 | 1 | 00 | 1
        ↓
11 | 000000 | 1
```

第二步：把这个被 `1` 包围的 `0` 段变成 `1`

```text
11 | 000000 | 1
        ↓
11 | 111111 | 1
```

去掉补上的左右两个 `1`，原字符串变成：

```text
1111111
```

原来 `1` 的数量是 `2`，左右两个 `0` 段长度是 `3 + 2 = 5`，所以答案是：

```text
2 + 5 = 7
```

## 图解二：段结构抽象

任何有效操作都长这样：

```text
... 1 | 000...000 | 111...111 | 000...000 | 1 ...
        zeroLeft      oneBlock     zeroRight
```

操作后：

```text
... 1 | 111...111111...111111...111 | 1 ...
        zeroLeft + oneBlock + zeroRight
```

中间 `oneBlock` 原来就是 `1`，最终还是 `1`。

真正额外变成 `1` 的只有：

```text
zeroLeft + zeroRight
```

所以题目不需要真的模拟两次翻转，只需要找：

```text
两个相邻 0 段长度之和的最大值
```

## 方法一：暴力枚举并模拟操作

### 思路

为了直接理解题意，可以暴力模拟所有可能的第一步：

1. 构造 `t = "1" + s + "1"`。
2. 枚举每一段连续的 `1`。
3. 判断这段 `1` 是否左右都是 `0`。
4. 如果合法，先把这段 `1` 改成 `0`。
5. 再枚举所有被 `1` 包围的 `0` 段，把其中一段改成 `1`。
6. 统计原字符串范围内的 `1` 数量，更新最大值。

这个方法适合帮助理解题目，但不适合提交。

### C++ 代码

```cpp
class Solution {
public:
    int maxActiveSectionsAfterTrade(string s) {
        string t = "1" + s + "1";
        int n = (int)t.size();

        auto countOriginalOnes = [&](const string& cur) {
            int cnt = 0;
            for (int i = 1; i + 1 < (int)cur.size(); ++i) {
                if (cur[i] == '1') ++cnt;
            }
            return cnt;
        };

        int ans = countOriginalOnes(t);

        for (int l = 1; l < n - 1; ++l) {
            if (t[l] != '1') continue;

            int r = l;
            while (r + 1 < n && t[r + 1] == '1') ++r;

            if (t[l - 1] == '0' && t[r + 1] == '0') {
                string afterFirst = t;
                for (int i = l; i <= r; ++i) {
                    afterFirst[i] = '0';
                }

                for (int a = 1; a < n - 1; ++a) {
                    if (afterFirst[a] != '0') continue;

                    int b = a;
                    while (b + 1 < n && afterFirst[b + 1] == '0') ++b;

                    if (afterFirst[a - 1] == '1' && afterFirst[b + 1] == '1') {
                        string afterSecond = afterFirst;
                        for (int i = a; i <= b; ++i) {
                            afterSecond[i] = '1';
                        }
                        ans = max(ans, countOriginalOnes(afterSecond));
                    }

                    a = b;
                }
            }

            l = r;
        }

        return ans;
    }
};
```

### 复杂度

- 时间复杂度：最坏约 `O(n^3)`。
- 空间复杂度：`O(n)`。

这个方法只是辅助理解，不推荐提交。

## 方法二：分组统计所有连续段

### 思路

把字符串按连续相同字符分组。

例如：

```text
s = "0011100110"

分组：
00 | 111 | 00 | 11 | 0
```

其中 `0` 段长度依次是：

```text
2, 2, 1
```

可以选择的结构是：

```text
0段 + 1段 + 0段
```

所以只需要枚举相邻的两个 `0` 段：

```text
2 + 2 = 4
2 + 1 = 3
```

最大新增量是 `4`。

原本有 `5` 个 `1`，所以答案是：

```text
5 + 4 = 9
```

### 图解

```text
s:        00 | 111 | 00 | 11 | 0
0段编号:   A          B        C

可选：
A + B = 2 + 2 = 4
B + C = 2 + 1 = 3

bestGain = 4
answer = 原有 1 数量 5 + 4 = 9
```

### C++ 代码

```cpp
class Solution {
public:
    int maxActiveSectionsAfterTrade(string s) {
        int ones = 0;
        vector<int> zeroGroups;

        for (int i = 0, n = (int)s.size(); i < n; ) {
            int j = i;
            while (j < n && s[j] == s[i]) {
                ++j;
            }

            int len = j - i;
            if (s[i] == '1') {
                ones += len;
            } else {
                zeroGroups.push_back(len);
            }

            i = j;
        }

        int bestGain = 0;
        for (int i = 1; i < (int)zeroGroups.size(); ++i) {
            bestGain = max(bestGain, zeroGroups[i - 1] + zeroGroups[i]);
        }

        return ones + bestGain;
    }
};
```

### 复杂度

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(n)`，最坏情况下字符串交替出现，`0` 段数量可以达到 `O(n)`。

## 方法三：贪心 + 双指针，一次遍历，O(1) 空间

### 思路

方法二存下了所有 `0` 段。

其实没有必要保存全部，只需要在遍历当前 `0` 段时记住上一个 `0` 段长度。

变量含义：

- `ones`：原字符串中 `1` 的数量；
- `previousZeroLen`：上一个连续 `0` 段的长度；
- `bestGain`：目前找到的最大新增量。

每遇到一个新的 `0` 段，设长度为 `curZeroLen`，就可以尝试：

```text
previousZeroLen + curZeroLen
```

然后更新：

```text
previousZeroLen = curZeroLen
```

### 为什么这个贪心是正确的

一次有效 trade 必须先选择一段被 `0` 包围的 `1`。

这说明它在原字符串的段结构里一定夹在两个 `0` 段中间：

```text
0段 | 1段 | 0段
```

最终净增加的一定是这两个 `0` 段的长度之和。

因此，所有可能操作对应的候选值就是：

```text
每一对相邻 0 段长度之和
```

我们遍历所有段时，每遇到一个 `0` 段，都和上一个 `0` 段组成一对，所有候选都会被检查到。

所以取最大值即可。

### C++ 代码

```cpp
class Solution {
public:
    int maxActiveSectionsAfterTrade(string s) {
        int ones = 0;
        int bestGain = 0;
        int previousZeroLen = -1000000000;

        for (int i = 0, n = (int)s.size(); i < n; ) {
            int j = i;
            while (j < n && s[j] == s[i]) {
                ++j;
            }

            int len = j - i;
            if (s[i] == '1') {
                ones += len;
            } else {
                bestGain = max(bestGain, previousZeroLen + len);
                previousZeroLen = len;
            }

            i = j;
        }

        return ones + bestGain;
    }
};
```

### 复杂度

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(1)`。

推荐提交这个方法。

## 边界情况

### 只有一个字符

```text
s = "0" -> 0
s = "1" -> 1
```

### 全是 0

```text
s = "00000"
```

没有可以先清零的 `1` 段，答案是 `0`。

### 全是 1

```text
s = "11111"
```

没有 `0` 段可以变成 `1`，答案是 `5`。

### 只有一个 0 段

```text
s = "00111"
```

虽然有 `0`，但不存在两个 `0` 段夹住一个 `1` 段，因此没有收益。

答案是原本 `1` 的数量，也就是 `3`。

## 常见错误

### 错误一：把操作理解成随便翻转一段 0

不能随便选择 `0` 段。

第二步选择的 `0` 段必须被 `1` 包围，而且这个 `0` 段通常是第一步清零后合并出来的大段。

### 错误二：忘记两端补 `1`

补 `1` 会影响边界处 `0` 段能不能在第二步被包围。

例如：

```text
s = "0100"
t = "101001"
```

右侧 `00` 因为后面有补上的 `1`，才能参与合并后的翻转。

### 错误三：误以为中间 `1` 段长度也要加入收益

中间 `1` 段本来就是活跃的，最终也还是活跃的。

真正新增的是左右两侧原本的 `0`。

所以收益不是：

```text
a + b + c
```

而是：

```text
a + c
```

### 错误四：没有处理不存在两个 0 段的情况

如果 `0` 段少于两个，就无法形成：

```text
0段 | 1段 | 0段
```

此时 `bestGain = 0`，答案就是原本 `1` 的数量。

## 最终推荐代码

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int maxActiveSectionsAfterTrade(string s) {
        int ones = 0;
        int bestGain = 0;
        int previousZeroLen = -1000000000;

        for (int i = 0, n = (int)s.size(); i < n; ) {
            int j = i;
            while (j < n && s[j] == s[i]) {
                ++j;
            }

            int len = j - i;
            if (s[i] == '1') {
                ones += len;
            } else {
                bestGain = max(bestGain, previousZeroLen + len);
                previousZeroLen = len;
            }

            i = j;
        }

        return ones + bestGain;
    }
};
```

## 可以学到的知识和技能

### 1. 字符串分段思想

很多二进制字符串题不应该逐个字符硬模拟，而应该先看连续段：

```text
0011100110 -> 00 | 111 | 00 | 11 | 0
```

分段之后，题目结构会清楚很多。

### 2. 把复杂操作转化成净收益

这题表面上是两次翻转：

```text
1 段 -> 0 段
0 段 -> 1 段
```

但本质上只需要关心：

```text
新增了多少个 1
```

这就是「从过程模拟」转化为「收益计算」。

### 3. 贪心枚举候选

每一次合法操作都对应一对相邻 `0` 段。

所以全局最优就是：

```text
max(相邻两个 0 段长度之和)
```

这是一种典型贪心：完整枚举所有候选收益，然后取最大。

### 4. 双指针扫描连续段

代码中：

```cpp
int j = i;
while (j < n && s[j] == s[i]) ++j;
```

这是常见的连续段扫描模板。

它可以用于：

- 压缩字符串；
- 统计连续相同字符；
- 处理区间段问题；
- run-length encoding，简称 RLE。

### 5. 边界补位思想

题目中的：

```text
t = "1" + s + "1"
```

是一种边界哨兵技巧。

它让边界处的判断更统一，尤其是判断某段是否被左右字符包围。

不过本题最终代码不必真的构造 `t`，因为经过推导后，只需要统计原字符串里的相邻 `0` 段。

## 总结

本题最重要的不是模拟 trade，而是看懂 trade 的净效果。

如果选择：

```text
0^a 1^b 0^c
```

最终净增加：

```text
a + c
```

所以答案就是：

```text
原本 1 的数量 + 相邻两个 0 段长度之和的最大值
```

推荐使用一次遍历的贪心写法：

- 时间复杂度 `O(n)`；
- 空间复杂度 `O(1)`；
- 代码短，逻辑稳定，适合提交。
